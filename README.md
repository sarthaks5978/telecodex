## 🔍 What is telecodex?

telecodex is a tool that connects your local Codex command-line interface (CLI) to Telegram. This lets you run Codex commands remotely, using Telegram messages. It keeps conversations organized by topic, supports file attachments, and shows replies as they stream in. This makes it simple to work with Codex from anywhere without needing to use your computer directly.

---

## 💻 System Requirements

To use telecodex on Windows, make sure your computer meets these minimum specs:

- Windows 10 or above (64-bit recommended)
- At least 4 GB of RAM
- 200 MB of free disk space
- Internet connection for Telegram messages
- Installed [Telegram Desktop](https://desktop.telegram.org/) or Telegram Mobile app

telecodex uses your existing Codex CLI installation. Make sure you have Codex CLI installed and tested locally before setting up telecodex.

## ⚙️ Using telecodex

After setup, you can start sending Codex commands via Telegram.

- Open Telegram and find the telecodex bot or chat.
- Send commands as normal messages.
- telecodex processes your requests, runs them locally using Codex CLI, and sends back the replies.
- Sessions are grouped by topics to keep conversations clear.
- Attach files in your Telegram messages to include them in Codex requests.
- Replies arrive in streaming form, showing results as they happen.

## 🔧 Troubleshooting

If telecodex does not work as expected, try these steps:

- Confirm your internet connection is stable.
- Check that Telegram is working on your device.

- Restart telecodex and try the Telegram link process again.

## 📋 Common Questions

**Can I use telecodex without the Telegram app?**  
No. telecodex relies on Telegram to send and receive commands and replies.

**Is Codex CLI included?**  
No. You need to install Codex CLI separately before using telecodex.

**Can I use this on macOS or Linux?**  
This version is for Windows only. Other platforms might be supported in the future.

**Are my messages private?**  
Yes. telecodex only accesses your Telegram messages and your local Codex CLI. It does not send data to external servers.
